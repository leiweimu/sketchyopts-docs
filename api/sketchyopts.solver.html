Nyström PCG#

sketchyopts.solver.nystrom_pcg(A, b, mu, rank, key, *, x0=None, tol=1e-05, maxiter=None)[source]#

The Nyström preconditioned conjugate gradient method (Nyström PCG).

The function solves the regularized linear system \((A + \mu I)x = b\) using Nyström PCG.

Nyström PCG uses randomized Nyström preconditioner by implicitly applying

\[P^{-1} = (\hat{\lambda}_{l} + \mu) U (\hat{\Lambda} + \mu I)^{-1} U^{T} + (I - U U^{T}) \]

where \(U\) and \(\hat{\Lambda}\) are from rank-\(l\) randomized Nyström approximation (here \(\hat{\lambda}_{l}\) is the \(l\)th diagonal entry of \(\hat{\Lambda}\)).

Nyström PCG terminates if the \(\ell_2\)-norm of the residual \(b - (A + \mu I)\hat{x}\) is within the specified tolerance or it has reached the maximal number of iterations.

References

  • Z. Frangella, J. A. Tropp, and M. Udell, Randomized Nyström preconditioning. SIAM Journal on Matrix Analysis and Applications, vol. 44, iss. 2, 2023, pp. 718-752.

Parameters:

  • A (Union[Array, ndarray, bool, number, bool, int, float, complex]) – A two-dimensional array representing a positive-semidefinite matrix.

  • b (Union[Array, ndarray, bool, number, bool, int, float, complex]) – A vector or a two-dimensional array giving the righthand side(s) of the regularized linear system.

  • mu (float) – Regularization parameter (with non-negative value).

  • rank (int) – Rank of the randomized Nyström approximation (which coincides with sketch size).

  • key (Union[Array, ndarray, bool, number, bool, int, float, complex]) – A PRNG key used as the random key.

  • x0 (Union[Array, ndarray, bool, number, bool, int, float, complex, None]) – Initial guess for the solution (same size as righthand side(s) b; default None). When set to None, the algorithm uses zero vector as starting guess.

  • tol (float) – Solution tolerance (default \(10^{-5}\)).

  • maxiter (Optional[int]) – Maximum number of iterations (default None). When set to None, the algorithm only terminates when the specified tolerance has been achieved. Internally the value gets set to ten times the size of the system.

Return type:

tuple[Array, Array, Array, int]

Returns:

A four-element tuple containing

  • x – Approximate solution to the regularized linear system. Solution has the same size as righthand side(s) b.

  • r – Residual of the approximate solution. Residual has the same size as righthand side(s) b.

  • status – Whether or not the approximate solution has converged for each righthand side. Status has the same size as the number of righthand side(s).

  • k – Total number of iterations to reach to the approximate solution.

SketchySGD#

class sketchyopts.solver.SketchySGD(*, fun, grad_fun=None, hvp_fun=None, pre_update=None, rank, rho, grad_batch_size, hess_batch_size, seed, learning_rate=0.5, maxiter=20, tol=0.001, verbose=False, update_freq)[source]#

The SketchySGD optimizer.

SketchySGD is a stochastic quasi-Newton method that uses sketching to approximate the curvature of the loss function. It maintains a preconditioner for SGD using randomized low-rank Nyström approximations to the subsampled Hessian and automatically selects an appropriate learning whenever it updates the preconditioner.

Example

from sketchyopts.solver import SketchySGD

def ridge_reg_objective(params, l2reg, data):
    X, y = data[:,:sample_dim], data[:,sample_dim:]
    residuals = jnp.dot(X, params) - y
    return jnp.mean(residuals ** 2) + 0.5 * l2reg * jnp.dot(w ** 2)

solver = SketchySGD(ridge_reg_objective, ...)
solver.run(init_params, data, l2reg=l2reg)

References

  • Z. Frangella, P. Rathore, S. Zhao, and M. Udell, SketchySGD: Reliable Stochastic Optimization via Randomized Curvature Estimates.

Parameters:

  • fun (Callable) – Objective function. The function needs to have the optimization variable as its first argument.

  • grad_fun (Optional[Callable]) – Optional gradient oracle that is expected to have the same function signature as the objective.

  • hvp_fun (Optional[Callable]) – Optional hessian-vector-product oracle that takes optimization variable params as its first argument, vector v as its second arguments.

  • pre_update (Optional[Callable]) – Optional gradient update function that gets called before every parameter update.

  • rank (int) – Rank of the preconditioner.

  • rho (float) – Regularization parameter (with non-negative value).

  • update_freq (int) – A non-negative integer that specifies the update frequency of the preconditioner. When set to 0 or \(\infty\) (e.g. jax.numpy.inf or numpy.inf), the optimizer uses constant preconditioner that is constructed at the beginning of the optimization process.

  • seed (int) – An integer used as a seed to generate random numbers.

  • learning_rate (Union[float, Callable]) – Step size for applying updates (default \(0.5\)). It can either be a fixed scalar value or a schedule based on step count. If a fixed scalar value is provided, the algorithm uses the value as the scaling factor to the adaptively chosen learning rate whenever the preconditioner is updated.

  • maxiter (int) – Maximum number of iterations to run the solver (default \(20\)).

  • tol (float) – Threshold of the gradient norm used for terminating the solver (default \(1e-3\)).

  • verbose (bool)
