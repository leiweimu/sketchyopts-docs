class sketchyopts.sketching.Sketching(sketch_size, dim, seed, mode='left')[source]#

Abstract class for sketching matrix.

The class defines the interface for a sketching matrix. The class represents a sketching matrix that can be applied to a linear operator or a vector. For a left-sketching matrix, the size is \((\text{sketch\_size}, \text{dim})\); for a right-sketching matrix, the size is \((\text{dim}, \text{sketch\_size})\).

Parameters:

  • sketch_size (int) ‚Äì Size of the sketch.

  • dim (int) ‚Äì Dimension of the input or output space for a left- or right-sketching matrix respectively.

  • key ‚Äì Seed for randomization.

  • mode (str) ‚Äì Sketching mode (default left). The mode can be either left for left-sketching or right for right-sketching.

class sketchyopts.sketching.GaussianEmbedding(sketch_size, dim, seed, mode='left')[source]#

Gaussian random matrix.

This sketching is defined by a Gaussian random matrix where each entry is independently drawn from a normal distribution with zero mean and variance \(1 / \text{sketch\_size}\).

class sketchyopts.sketching.OrthonormalEmbedding(sketch_size, dim, seed, mode='left')[source]#

Orthonormal random matrix.

This sketching is defined by an orthonormal random matrix where each column is an orthonormal basis.

class sketchyopts.sketching.SJLT(sketch_size, dim, seed, mode='left', sparsity=None)[source]#

Sparse Johnson‚ÄìLindenstrauss transform (SJLT) matrix.

This sketching is defined by a sparse Johnson‚ÄìLindenstrauss transform (SJLT) matrix where each row has exactly \(\zeta\) non-zero entries. Each non-zero entry is independently drawn from a Rademacher distribution (uniform over \(\{\pm 1\}\)), and its location in the row is randomly chosen from \(\{1, \cdots, \text{dim}\}\).

The sketching has an additional optional argument sparsity that specifies the value of \(\zeta\). By default (None), the sparsity parameter \(\zeta\) takes the recommended practical value of \(\operatorname{min}\{8, \text{sketch\_size}\}\) [1].

References

[1]

J. A. Tropp, A. Yurtsever, M. Udell, and V. Cevher, Streaming Low-Rank Matrix Approximation with an Application to Scientific Simulation, SIAM Journal on Scientific Computing, 41(4), 2019.

class sketchyopts.sketching.SRTT(sketch_size, dim, seed, mode='left')[source]#

Subsampled randomized trigonometric transform (SRTT) sketching matrix.

This sketching is defined by a random restriction \(R\), a discrete cosine transform \(F\), a random sign flip \(E\), and a random permutation \(\Gamma\):

\[S = \sqrt{\frac{1}{\text{sketch\_size}}} R F E \Gamma \]

Here random random restriction selects uniformly \(\text{sketch\_size}\) random entries from its input; sign flip is a diagonal matrix such that its diagonal entries are i.i.d Rademacher random variables (i.e. random variable with equal probability mass assigned to \(\{\pm 1\}\)); random permutation is a square matrix drawn uniformly at random from the set of \(\text{dim}\)-permutation matrices.

class sketchyopts.sketching.SRHT(sketch_size, dim, seed, mode='left')[source]#

Subsampled randomized Hadamard transform (SRHT) sketching matrix.

This sketching is defined by a random restriction \(R\), a Hadamard transform \(H\), a random sign flip \(E\):

\[S = \sqrt{\frac{1}{\text{sketch\_size}}} R H E \]

Here random random restriction selects uniformly \(\text{sketch\_size}\) random entries from its input; sign flip is a diagonal matrix such that its diagonal entries are i.i.d Rademacher random variables (i.e. random variable with equal probability mass assigned to \(\{\pm 1\}\)).

For a matrix with dimension \(\text{dim}\) that is not a power of two (required for defining a Hadamard matrix), sketching is instead applied to a padded version of the matrix, where additional zero rows are appended.

class sketchyopts.sketching.LESS(sketch_size, dim, seed, mode='left', compute_leverage_scores=False, matrix=None)[source]#

Leverage score sparsified embedding sketching matrix.

Suppose the matrix to be sketched is of size \(\text{dim} \times m\) with leverage scores \((l_1, \cdots, l_{\text{dim}})\). Let \(s_i \sim \mathrm{Multinomial}(m, p_1, \cdots, p_{\text{dim}})\) for \(i = 1, \cdots, \text{sketch\_size}\) be i.i.d. multinomial random variables with probability \(p_j\) approximately proportional to \(l_j\) for \(j = 1, \cdots, \text{dim}\).

Each row of the sketching matrix is independently determined by:

\[s_i^{\mathsf{T}} = \sqrt{\frac{1}{\text{sketch\_size}}} \bigg( \sqrt{\frac{s_{i,1}}{p_1}} x_{i,1}, \cdots, \sqrt{\frac{s_{i,\text{dim}}}{p_{\text{dim}}}} x_{i,\text{dim}} \bigg) \]

where \(x_{i,j}\)‚Äôs are i.i.d. mean zero and unit variance sub-Gaussian random variables (the implementation uses Rademacher random variables).

The sketching has two additional optional arguments:

  • compute_leverage_scores specifies the approach to construct LESS embedding (default False). If set to True, then leverage scores are approximated [2]. Otherwise the matrix gets transformed first so that leverage scores are approximately uniform.

  • matrix specifies the matrix to be sketched (default None). If not provided, the sketching matrix construction is deferred when matrix_sketch gets called.
